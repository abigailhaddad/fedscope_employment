Overview

Important: This documentation is created by an independent researcher based on publicly available FedScope data. For official documentation and authoritative information, please visit OPM's official FedScope website.

The FedScope Employment Cube contains snapshots of federal civilian employment data collected quarterly by the U.S. Office of Personnel Management (OPM). This dataset provides comprehensive information about federal employees including demographics, job characteristics, compensation, and organizational details.

Coverage

  • Time Period: 1998 - 2024 (quarterly snapshots)
  • Population: Federal civilian employees
  • Exclusions: Military personnel, intelligence agencies, postal service employees
  • Total Records: Approximately 140+ million employee records across 72 quarterly datasets
Privacy Note: All data is anonymized. Individual employees cannot be identified from the dataset.

Why Use This Dataset

The Challenge with Original FedScope Data

Working with the original FedScope data from OPM presents several challenges:

  • Manual Processing Required: Each quarterly dataset is a separate ZIP file containing multiple text files that must be downloaded, extracted, and merged manually
  • Complex Joins: Each ZIP contains one fact table and 13+ lookup tables that must be joined using proper keys
  • Duplicate Lookup Entries: Earlier datasets (1998-2003) contain duplicate entries in lookup tables
  • Schema Evolution: The data structure changes over time (e.g., pay plan field added in 2016)

What This Processed Dataset Provides

  • Ready-to-Use Format: All 72 quarterly datasets are already extracted, cleaned, and denormalized
  • Single Table per Quarter: Each CSV file contains all fields with human-readable descriptions already joined
  • Duplicate Resolution: Lookup table duplicates have been resolved consistently (using first occurrence within each dataset)
  • Consistent Schema: Schema changes are handled transparently with appropriate null values where fields don't exist
  • No Technical Barriers: Download and analyze immediately without needing to write merge code or handle edge cases
Note: What typically takes hours of data processing work is ready to use immediately.

Coding Schemes

Age Levels (agelvl)

Age is reported in 5-year bands to protect privacy:

  • A: Less than 20 years
  • B: 20-24 years
  • C: 25-29 years
  • D: 30-34 years
  • E: 35-39 years
  • F: 40-44 years
  • G: 45-49 years
  • H: 50-54 years
  • I: 55-59 years
  • J: 60-64 years
  • K: 65 years and over

Education Levels (edlvl)

Education codes range from 00 to 22:

  • 00-05: Less than high school
  • 06-11: High school diploma/GED
  • 12-14: Some college
  • 15: Bachelor's degree
  • 16-17: Some graduate work
  • 18: Master's degree
  • 19-21: Professional/Doctoral degree
  • 22: Post-doctoral

PATCO Categories

  • P: Professional - Requires knowledge in specialized field (engineers, scientists, doctors)
  • A: Administrative - Management or staff functions (HR, budget, procurement)
  • T: Technical - Support to professional/administrative (technicians, assistants)
  • C: Clerical - Structured work support (data entry, filing)
  • O: Other - Trades, crafts, labor, protective services

Major Pay Plans

  • GS: General Schedule (most common)
  • ES: Senior Executive Service
  • SL/ST: Senior Level/Scientific & Technical
  • AD: Administratively Determined
  • FW: Federal Wage System

Data Quality Notes

Schema Evolution

  • The pp (pay plan) field was added in 2016
  • Some fields may have null or blank values in earlier years
  • Agency codes and names may change over time due to reorganizations

Known Issues

Within-Dataset Duplicate Agencies (1998-2003): Within individual quarterly datasets, some agencies appear with multiple names but the same code due to reorganizations. The processing pipeline uses the first occurrence of each code within each dataset. See the complete list of duplicates in lookup_duplicates_summary.txt.
Redacted Salary Values: Some salary values in the original data are masked as '****' for privacy reasons. These are replaced with null values in the processed dataset.

Data Coverage

  • 1998-2008: Annual September snapshots only
  • 2009: September and December
  • 2010-2024: Full quarterly coverage (March, June, September, December)
  • 2024: Data through September
